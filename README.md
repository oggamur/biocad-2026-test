# Biocad 2026 Test Project

Веб-приложение с дашбордами устройств.

## Технологии

- Node.js
- Express.js
- Нативный JavaScript
- HTML5
- SCSS (Sass)

## Структура проекта

```
biocad-2026-test/
├── server.js              # Node.js сервер
├── package.json           # Зависимости проекта
├── links.json            # Файл с ссылками на деплой и код
├── scss/                 # SCSS исходники
│   ├── _variables.scss  # Переменные (цвета, типографика, отступы)
│   ├── _mixins.scss     # Миксины (переиспользуемые стили)
│   ├── _base.scss       # Базовые стили
│   ├── _header.scss     # Стили шапки
│   ├── _layout.scss     # Стили layout
│   ├── _device-card.scss # Стили карточек устройств
│   ├── _device-detail.scss # Стили детального просмотра
│   ├── _error.scss      # Стили страницы ошибки
│   ├── _responsive.scss # Адаптивные стили
│   └── main.scss        # Главный файл (импортирует все модули)
├── public/               # Статические файлы
│   ├── dashboards.html  # Страница дашбордов
│   ├── device-detail.html # Страница детального просмотра
│   ├── error.html       # Страница ошибки
│   ├── css/
│   │   └── styles.css   # Скомпилированные стили (генерируется из SCSS)
│   ├── images/
│   │   └── devices/     # Изображения устройств (см. README.md в папке)
│   └── js/
│       ├── dashboards.js # Логика дашбордов
│       └── device-detail.js # Логика детального просмотра

```

## Установка и запуск

1. Установите зависимости:
```bash
npm install
```

2. Скомпилируйте SCSS в CSS:
```bash
npm run build:css
```

Или для автоматической компиляции при изменениях:
```bash
npm run watch:css
```

3. Запустите сервер:
```bash
npm start
```

4. Откройте в браузере:
```
http://localhost:3000
```

## Работа со стилями (SCSS)

Стили организованы в модульную структуру SCSS:

- **`_variables.scss`** - все переменные (цвета, размеры шрифтов, отступы)
- **`_mixins.scss`** - переиспользуемые миксины (flexbox, типографика, кнопки)
- **`_base.scss`** - базовые стили (reset, body)
- **`_header.scss`** - стили шапки сайта
- **`_layout.scss`** - стили layout (сетка, контейнеры)
- **`_device-card.scss`** - стили карточек устройств
- **`_device-detail.scss`** - стили страницы детального просмотра
- **`_error.scss`** - стили страницы ошибки
- **`_responsive.scss`** - адаптивные стили (медиа-запросы)

Все модули импортируются в **`main.scss`**, который компилируется в `public/css/styles.css`.

## Функциональность

### Страницы

- **Дашборды** (`/dashboards`) - отображение карточек устройств в виде сетки
- **Ошибка** (`/error`) - страница обработки ошибок с возможностью возврата к дашбордам

### API Endpoints

- `GET /api/devices` - получение списка устройств
- `POST /api/devices/:id/state` - изменение состояния устройства

### Обработка ошибок

При ошибках загрузки данных происходит автоматический переход на страницу ошибки.

## Изображения устройств

В проекте используется система изображений для устройств. Изображения должны быть размещены в папке `public/images/devices/`:

- `ventilation.png` - Рециркуляционная вентиляция
- `osmometer.png` - Осмометр
- `alkali.png` - Система щелочи
- `biosafety.png` - Бокс биологической безопасности

Если изображение не найдено, будет использован fallback (эмодзи). Подробнее см. `public/images/devices/README.md`.

## Деплой

После деплоя на GitHub Pages или другой платформе, обновите файл `links.json` с актуальными ссылками:
- `deployment` - ссылка на развернутое приложение
- `code` - ссылка на репозиторий с кодом
